## 1. 系统概述与设计理念

### 1.1 系统定位

本系统是一个面向老年人的**多模态认知训练与健康监护平台**，融合了：
- **认知训练**：基于神经科学原理的认知刺激任务
- **健康监护**：非接触式生理信号监测（心率、呼吸率）
- **智能交互**：语音唤醒与自然语言交互
- **情感关怀**：基于情绪识别的个性化反馈

### 3.2 模块职责划分

| 模块 | 职责 | 技术实现 |
|------|------|----------|
| **Frontend** | 用户界面展示、交互处理 | Vue 3 + TypeScript |
| **VoiceManager** | 语音唤醒、识别、合成 | Sherpa ASR + VITS |
| **GameManager** | 游戏生命周期、难度调整 | 自定义游戏引擎 |
| **HealthMonitor** | 生理信号监测、健康评估 | rPPG + OpenCV |
| **EmergencyHandler** | 紧急呼救、安全响应 | WebSocket + 电话接口 |
| **LLM Engine** | 自然语言理解、对话生成 | Ollama + Gemma |

## 附录：关键设计决策

### A.1 技术选型理由

| 组件 | 选型 | 理由 |
|------|------|------|
| **前端框架** | Vue 3 | 响应式、轻量、生态成熟 |
| **后端框架** | FastAPI | 高性能、异步支持、自动文档 |
| **语音识别** | Sherpa | 轻量、可离线运行 |
| **语音合成** | VITS | 高质量、多说话人支持 |
| **LLM** | Ollama | 本地部署、隐私保护 |
| **数据库** | SQLite | 轻量、无需额外服务 |

### A.2 安全设计原则

1. **数据隐私**：所有数据本地存储，不上传云端
2. **紧急响应**：多重安全检查，确保紧急呼救可靠
3. **系统稳定性**：异常处理与自动恢复机制
4. **用户保护**：避免过度训练，保护身体健康

### A.3 性能优化策略

1. **前端缓存**：静态资源缓存、数据缓存
2. **后端优化**：异步处理、数据库索引
3. **网络优化**：WebSocket长连接、消息压缩
4. **资源管理**：定时器清理、内存管理
