# 语音交互卡顿 & 重复唤醒问题诊断与修复

## 一、问题根因分析

### 🔴 重复唤醒（最严重）

```
时间线：
0.0s  用户说"阿康阿康" → 触发唤醒
0.1s  状态变为 RESPONDING，播放"来啦"
0.2s  ❌ is_playing 在 play_thread 中异步设置，存在竞态窗口
      → 麦克风已采集到"来啦"的音频 → 放入 audio_queue
0.5s  KWS 从队列中处理残留音频 → 可能误检（或"来啦"的共振触发）
1.0s  ❌ 第二次唤醒触发！
```

**具体原因：**

| #  | 问题                                                                          | 位置                            |
| :- | :-------------------------------------------------------------------------- | :---------------------------- |
| 1  | `is_playing` 在 `play_thread` 内部才设为 `True`，而 `_audio_callback` 在主线程运行，存在竞态窗口 | `_speak` vs `_audio_callback` |
| 2  | 队列中已有音频块不会被移除，`is_playing=True` 后队列中的残留音频仍会被 KWS 处理                         | `_process_audio`              |
| 3  | STANDBY 状态下防抖仅 0.8s，太短                                                      | `_on_keyword_detected`        |
| 4  | 播放 LLM 回复时，如果回复内容包含"阿康"或相似音，麦克风拾取后再次触发                                      | 全流程                           |
| 5  | `_on_keyword_detected` 对非 STANDBY 状态的防抖是 1.5s，但**会话结束回到 STANDBY 后防抖就失效了**   | `_on_keyword_detected`        |

### 🔴 卡顿（延迟高）

```
当前延迟构成：
唤醒 → 播放"来啦"(1-2s) → speak_event.wait(3s) → sleep(0.3s) → sleep(0.5s) → sleep(0.2s) → 开始录音
总计：5-6秒 从唤醒到可以说话！
```

| #  | 问题                                        | 浪费时间     |
| :- | :---------------------------------------- | :------- |
| 1  | `speak_event.wait(timeout=3.0)` 不必要的3秒等待  | 3s       |
| 2  | 三个硬编码 `time.sleep` 累加                     | 1s       |
| 3  | 轮询 `is_playing` 每0.1s，响应慢                 | 0.1-0.2s |
| 4  | 前后端可能双重播报（`voice_speaking` + 后端 `_speak`） | 1-2s     |
| 5  | LLM 调用同步阻塞，无流式输出                          | 5-15s    |

***

## 二、修复方案

### 修复1：解决重复唤醒 — 播放期间彻底静默 KWS

```python
# voice_manager_sherpa.py
class VoiceManager:
    # ...
    
    # ⭐ 新增：KWS暂停控制
    self._kws_paused = False
    self._kws_lock = threading.Lock()
    
    def _audio_callback(self, indata, frames, time_info, status):
        """麦克风录音回调 - 修复竞态条件"""
        if status:
            print(f"[VoiceManager] 音频状态警告: {status}")
        
        # ⭐ 优先检查 _kws_paused（线程安全标记）
        # 不再依赖 is_playing（它在线程中异步设置，有竞态）
        kws_should_skip = False
        with self._kws_lock:
            kws_should_skip = self._kws_paused
        
        if not kws_should_skip:
            # 只在非暂停时放入KWS队列
            self.audio_queue.put(indata.copy())
        
        # 录音数据始终保存（录音状态由 is_recording 控制）
        if self.is_recording:
            self.recorded_audio.append(indata.copy())
            self._process_vad(indata)
    
    def _pause_kws(self):
        """暂停KWS检测（播放前调用）"""
        with self._kws_lock:
            self._kws_paused = True
        # ⭐ 清空队列中的残留音频
        while not self.audio_queue.empty():
            try:
                self.audio_queue.get_nowait()
            except queue.Empty:
                break
        print("[VoiceManager] KWS 已暂停，队列已清空")
    
    def _resume_kws(self):
        """恢复KWS检测（播放结束后延迟调用）"""
        with self._kws_lock:
            self._kws_paused = False
        print("[VoiceManager] KWS 已恢复")
```

### 修复2：延长防抖 + 会话级冷却

```python
def _on_keyword_detected(self, keyword):
    """检测到关键词时的回调 - 修复重复唤醒"""
    print(f"[VoiceManager] 检测到唤醒词: {keyword}")
    current_time = time.time()
    time_since_last_wake = current_time - self.last_wake_time
    
    # ⭐ 全局冷却：无论什么状态，2秒内不重复触发
    COOLDOWN_TIME = 2.0
    if time_since_last_wake < COOLDOWN_TIME:
        print(f"[VoiceManager] 冷却中: 忽略 ({time_since_last_wake:.2f}s < {COOLDOWN_TIME}s)")
        return
    
    # ⭐ 如果当前不在待机状态，不允许唤醒（除非超过5秒强制打断）
    if self.state != "STANDBY":
        if time_since_last_wake < 5.0:
            print(f"[VoiceManager] 非待机状态，忽略唤醒 (当前: {self.state})")
            return
        # 超过5秒的卡死会话，强制打断
        print(f"[VoiceManager] ⭐ 强制打断卡死会话: 当前状态 {self.state}")
        with self._session_lock:
            self._session_id += 1
            new_session_id = self._session_id
        self.is_recording = False
        self.is_playing = False
        self.recorded_audio = []
        self.vad_is_speaking = False
        self.vad_silence_start_time = None
        self.vad_speech_start_time = None
        self._pause_kws()
        time.sleep(0.1)
        self._trigger_wake_up(new_session_id)
        self.last_wake_time = current_time
        return
    
    # 正常唤醒
    self._trigger_wake_up()
    self.last_wake_time = current_time
```

### 修复3：消除卡顿 — 重写 `_start_recording_session`

```python
def _start_recording_session(self, response, session_id):
    """开始录音会话 - 优化版，消除多余延迟"""
    with self._session_lock:
        self._current_session_id = session_id
    
    def is_session_valid():
        with self._session_lock:
            return self._current_session_id == session_id
    
    if response:
        # ⭐ 1. 先暂停KWS，再播放（防止回声触发唤醒）
        self._pause_kws()
        
        self._set_state("RESPONDING")
        print(f"[VoiceManager] [会话{session_id}] 状态: RESPONDING -> {response}")
        
        # 通知前端
        self.socketio.emit('voice_wake_up', {
            'response': response,
            'user_text': '',
            'state': 'RESPONDING',
            'session_id': session_id
        })
        
        # ⭐ 2. 播放回应语（同步等待完成）
        self._speak_sync(response)  # 使用同步版本
        
        # ⭐ 3. 检查打断
        if not is_session_valid():
            return
        
        # ⭐ 4. 只保留必要延迟（等音频设备释放）
        time.sleep(0.3)  # 从1s减少到0.3s
        
        if not is_session_valid():
            return
    else:
        self._pause_kws()
    
    # ⭐ 5. 清空KWS队列（可能积累了播放期间的残留）
    while not self.audio_queue.empty():
        try:
            self.audio_queue.get_nowait()
        except queue.Empty:
            break
    
    # ⭐ 6. 延迟恢复KWS（等待回声完全消失）
    threading.Timer(0.5, self._resume_kws).start()
    
    # 进入聆听状态
    self._set_state("LISTENING")
    self.is_recording = True
    self.recorded_audio = []
    self.vad_is_speaking = False
    self.vad_silence_start_time = None
    self.vad_speech_start_time = None
    
    print(f"[VoiceManager] [会话{session_id}] 状态: LISTENING -> 开始录音")
    
    self.socketio.emit('voice_status', {
        'status': 'listening',
        'message': '正在听...',
        'state': 'LISTENING',
        'session_id': session_id
    })
    
    # 录音循环
    start_time = time.time()
    while self.is_recording and self.is_running:
        if not is_session_valid():
            self.is_recording = False
            return
        elapsed = time.time() - start_time
        if elapsed >= self.VAD_MAX_RECORD_DURATION:
            self.is_recording = False
            break
        time.sleep(0.05)
    
    self.is_recording = False
    
    if not is_session_valid():
        return
    
    if len(self.recorded_audio) > 0:
        total_samples = sum(len(chunk) for chunk in self.recorded_audio)
        duration = total_samples / self.SAMPLE_RATE
        print(f"[VoiceManager] [会话{session_id}] 录制时长: {duration:.1f} 秒")
        if duration >= self.VAD_MIN_SPEECH_DURATION:
            self._process_recorded_audio(session_id)
        else:
            print(f"[VoiceManager] [会话{session_id}] 语音太短")
            self._back_to_standby()
    else:
        self._back_to_standby()
```

### 修复4：添加同步播报方法 + 播报期间暂停KWS

```python
def _speak_sync(self, text):
    """同步语音合成并播放（阻塞直到完成）"""
    if not text:
        return
    
    # ⭐ 播报前暂停KWS
    self._pause_kws()
    self.is_playing = True
    
    try:
        if self.tts_engine_type == 'vits' and self.tts_engine:
            audio = self.tts_engine.generate(
                text, sid=self.tts_sid, speed=max(1.2, self.tts_speed)
            )
            samples = np.array(audio.samples, dtype=np.float32) * self.tts_volume
            sd.play(samples, samplerate=audio.sample_rate)
            sd.wait()  # ⭐ 同步等待播放完成
        elif self.tts_engine_type == 'pytts' and self.pytts_engine:
            import pyttsx3
            engine = pyttsx3.init()
            engine.setProperty('rate', int(150 * self.tts_speed))
            engine.setProperty('volume', min(1.0, self.tts_volume))
            engine.say(text)
            engine.runAndWait()
            engine.stop()
        else:
            # 尝试可用的引擎
            if self.tts_engine:
                audio = self.tts_engine.generate(text, sid=self.tts_sid, speed=max(1.2, self.tts_speed))
                samples = np.array(audio.samples, dtype=np.float32) * self.tts_volume
                sd.play(samples, samplerate=audio.sample_rate)
                sd.wait()
    except Exception as e:
        print(f"[VoiceManager] 同步播报错误: {e}")
    finally:
        self.is_playing = False
def _speak(self, text):
    """异步语音播报 - 用于LLM回复等场景，播报期间暂停KWS"""
    if not text:
        return
    
    with self._speaking_lock:
        self._current_speak_session += 1
        current_session = self._current_speak_session
    
    print(f"[VoiceManager] 语音播报 ({self.tts_engine_type}): {text} [会话{current_session}]")
    
    def play_thread():
        # ⭐ 播报前暂停KWS
        self._pause_kws()
        self.is_playing = True
        
        def is_speaking_session_valid():
            with self._speaking_lock:
                return self._current_speak_session == current_session
        
        try:
            if not is_speaking_session_valid():
                return
            
            if self.tts_engine_type == 'vits' and self.tts_engine:
                audio = self.tts_engine.generate(text, sid=self.tts_sid, speed=max(1.2, self.tts_speed))
                if not is_speaking_session_valid():
                    return
                samples = np.array(audio.samples, dtype=np.float32) * self.tts_volume
                sd.play(samples, samplerate=audio.sample_rate)
                # ⭐ 同步等待，但定期检查打断
                while sd.get_stream().active:
                    if not is_speaking_session_valid():
                        sd.stop()
                        return
                    time.sleep(0.05)
            elif self.tts_engine_type == 'pytts' and self.pytts_engine:
                import pyttsx3
                engine = pyttsx3.init()
                engine.setProperty('rate', int(150 * self.tts_speed))
                engine.setProperty('volume', min(1.0, self.tts_volume))
                engine.say(text)
                if not is_speaking_session_valid():
                    engine.stop()
                    return
                engine.runAndWait()
                engine.stop()
            else:
                if self.tts_engine:
                    audio = self.tts_engine.generate(text, sid=self.tts_sid, speed=max(1.2, self.tts_speed))
                    samples = np.array(audio.samples, dtype=np.float32) * self.tts_volume
                    sd.play(samples, samplerate=audio.sample_rate)
                    sd.wait()
        except Exception as e:
            print(f"[VoiceManager] 语音播放错误: {e}")
        finally:
            self.is_playing = False
            # ⭐ 播报结束后延迟恢复KWS
            threading.Timer(0.5, self._resume_kws).start()
            print(f"[VoiceManager] 播报完成 [会话{current_session}]")
    
    threading.Thread(target=play_thread, daemon=True).start()
```

### 修复6：LLM回复播报也需暂停KWS + 防止前后端双重播报

```python
def _call_llm_and_respond(self, user_text, session_id=None):
    """调用LLM并回应 - 优化版"""
    def is_session_valid():
        with self._session_lock:
            return self._current_session_id == session_id
    
    try:
        if session_id and not is_session_valid():
            return
        
        # 调用 LLM
        from core.core_agent import ask_akon
        system_state = self.system_core.get_state() if self.system_core else {}
        reply_text, action = ask_akon(user_text, system_state)
        
        if session_id and not is_session_valid():
            return
        
        if reply_text:
            print(f"[VoiceManager] [会话{session_id}] LLM回复: {reply_text}")
            
            # 通知前端显示文本（⭐ 不再让前端播放语音，由后端统一播放）
            self.socketio.emit('voice_llm_response', {
                'text': reply_text,
                'state': 'PROCESSING',
                'session_id': session_id
            })
            
            # ⭐ 执行页面跳转action
            if action and action.get('type') == 'navigate':
                page = action.get('page', '/')
                self.socketio.emit('navigate_to', {'page': page})
                print(f"[VoiceManager] [会话{session_id}] 跳转页面: {page}")
            
            # 后端统一播报（_speak内部已处理KWS暂停）
            self._speak(reply_text)
        else:
            self._speak("抱歉，我没听明白")
    except Exception as e:
        print(f"[VoiceManager] [会话{session_id}] LLM调用出错: {e}")
        self._speak("网络有点问题，请稍等")
    finally:
        # ⭐ 只有当前会话有效时才回到待机
        if not session_id or is_session_valid():
            # 延迟回到待机，等播报完成
            time.sleep(1.0)
            self._back_to_standby()
```

def _back_to_standby(self):
    """回到待机状态"""
    # ⭐ 确保KWS恢复
    self._resume_kws()
    self._set_state("STANDBY")
    print("[VoiceManager] 状态: STANDBY -> 等待唤醒...")
    
    self.socketio.emit('voice_status', {
        'status': 'standby',
        'message': '等待唤醒...',
        'state': 'STANDBY'
    })

## 四、核心原则总结

| 原则                                              | 说明                                       |
| :---------------------------------------------- | :--------------------------------------- |
| **播报前暂停KWS**                                    | `_pause_kws()` 必须在 `_speak()` **之前**同步调用 |
| **播报后延迟恢复KWS**                                  | 用 `Timer(0.5, _resume_kws)` 延迟恢复，等回声消失   |
| **暂停时清空队列**                                     | 防止残留音频块被KWS处理                            |
| **用** **`_kws_paused`** **代替** **`is_playing`** | `_kws_paused` 在主线程同步设置，无竞态               |
| **全局冷却时间**                                      | 无论什么状态，2秒内不接受新唤醒                         |
| **同步播报回应语**                                     | 回应语用 `_speak_sync()` 同步播放，避免复杂的异步等待逻辑    |
| **去除多余sleep**                                   | 只保留0.3s音频设备释放等待                          |
