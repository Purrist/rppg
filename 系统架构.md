# 老年陪伴机器人系统架构

## 1. 系统架构概述

本系统采用前后端分离的架构设计，结合多模态感知和智能交互技术，为老年人提供健康监测、认知训练和情感陪伴服务。系统整体分为前端、后端和硬件三个主要部分，通过网络通信实现数据交换和功能协同。

### 1.1 系统层次结构

```
┌─────────────────────────────────────────────────────────────────────────────────┐
│                                硬件层                                         │
├─────────────────────────────────────────────────────────────────────────────────┤
│ 平板设备（状态采集）  │  外置摄像头（地面交互）  │  投影仪（地面投影）         │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                后端层                                         │
├─────────────────────────────────────────────────────────────────────────────────┤
│ 感知模块  │  核心控制  │  游戏模块  │  语音模块  │  数据存储  │  大语言模型   │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                前端层                                         │
├─────────────────────────────────────────────────────────────────────────────────┤
│ 平板界面  │  投影界面  │  语音交互  │  数据可视化                           │
└─────────────────────────────────────────────────────────────────────────────────┘
```

## 2. 核心模块详细说明

### 2.1 前端架构

#### 2.1.1 技术栈
- **框架**：Nuxt.js + Vue 3
- **状态管理**：Pinia
- **通信**：Socket.io-client
- **UI组件**：自定义组件
- **样式**：CSS3

#### 2.1.2 页面结构

| 页面名称 | 路径 | 功能描述 |
|---------|------|----------|
| 主页 | / | 系统入口，显示天气、问候语和快捷功能 |
| 健康页面 | /health | 健康监测数据可视化和健康建议 |
| 娱乐页面 | /entertainment | 影视、音乐、舞蹈等娱乐功能 |
| 认知训练 | /learning | 认知训练游戏列表和训练记录 |
| 处理速度训练 | /processing-speed | 处理速度训练游戏界面 |
| 投影页面 | /projection | 投影交互界面控制 |
| 设置页面 | /settings | 系统设置和个性化配置 |
| 开发者页面 | /developer | 系统调试和监控 |

#### 2.1.3 核心组件

| 组件名称 | 功能描述 | 技术实现 |
|---------|----------|----------|
| GameFrame | 游戏框架组件 | Vue 3 组合式API |
| GameRules | 游戏规则展示 | 响应式布局 |
| ProcessingSpeedGame | 处理速度训练游戏 | Canvas 2D 绘图 |
| WhackAMole | 打地鼠游戏 | Canvas 2D 绘图 |
| systemStore | 全局状态管理 | Pinia |

### 2.2 后端架构

#### 2.2.1 技术栈
- **语言**：Python 3.8+
- **Web框架**：Flask
- **实时通信**：Flask-SocketIO
- **视频处理**：OpenCV
- **语音处理**：Sherpa-ai
- **状态管理**：自定义单例模式
- **数据存储**：JSON文件

#### 2.2.2 核心模块

##### 2.2.2.1 SystemCore（系统核心）
- **功能**：系统唯一的状态中心，管理所有系统状态
- **特点**：
  - 线程安全的单例模式
  - 状态变更自动广播
  - 数据持久化
  - 系统监控
- **核心状态**：
  - AI模式（基础/智伴）
  - 页面状态和流转
  - 游戏状态（待机/预备/游戏中/暂停/结算）
  - 游戏运行时数据（得分、时间、准确率）
  - 用户感知数据（人物、情绪、注意力、疲劳度、心率）
  - 环境数据
  - 用户偏好和历史

##### 2.2.2.2 感知模块
- **功能**：处理视频流数据，识别用户状态和环境信息
- **子模块**：
  - perception_manager.py：感知管理器
  - perception_screen_processor.py：屏幕处理器
  - utils.py：工具函数
- **感知内容**：
  - 人物检测和计数
  - 面部表情分析（情绪识别）
  - 身体姿态分析
  - 脚部位置检测（用于地面交互）
  - 心率估计（rPPG）
  - 环境光强检测

##### 2.2.2.3 游戏模块
- **功能**：管理认知训练游戏的逻辑和状态
- **子模块**：
  - games_manager.py：游戏管理器
  - game_whack_a_mole.py：打地鼠游戏
  - processing_speed_game.py：处理速度训练游戏
  - difficulty_adjuster.py：难度调节器
  - games_base.py：游戏基类
- **游戏类型**：
  - 打地鼠（反应速度训练）
  - 处理速度训练（Go/No-Go、选择反应、序列反应）
  - 色词测试（注意力训练）

##### 2.2.2.4 语音模块
- **功能**：处理语音输入和输出
- **子模块**：
  - voice_manager_sherpa.py：语音管理器
- **功能**：
  - 语音唤醒（"阿康阿康"）
  - 语音识别
  - 语音合成（TTS）
  - 语音状态管理

##### 2.2.2.5 数据存储
- **功能**：存储系统配置、训练历史和用户数据
- **存储结构**：
  - system_config.json：系统配置
  - training_history.json：训练历史
  - training_summary.json：训练摘要
  - session_*.json：游戏会话数据
- **数据备份**：
  - 配置文件备份
  - 历史数据备份

##### 2.2.2.6 大语言模型集成
- **功能**：提供智能对话和决策能力
- **集成方式**：本地运行 Qwen3.5
- **工具调用**：
  - 搜索图片、音频、视频
  - 生成游戏
  - 语音合成
  - 难度调整

### 2.3 硬件层

#### 2.3.1 设备清单

| 设备名称 | 功能描述 | 连接方式 |
|---------|----------|----------|
| 平板设备 | 状态采集（摄像头）、系统控制界面 | 局域网WiFi |
| 外置摄像头 | 地面交互检测 | USB连接 |
| 投影仪 | 地面投影界面 | HDMI连接 |
| 电脑主机 | 系统运行载体 | 本地运行 |

#### 2.3.2 设备配置
- **平板**：安卓平板，前置摄像头，通过IP Webcam应用将视频流传输到电脑
- **摄像头**：外置USB摄像头，用于拍摄地面投影区域
- **投影仪**：支持HDMI输入，投射到地面形成交互区域
- **网络**：局域网WiFi，确保设备间通信稳定

## 3. 数据流和交互流程

### 3.1 数据流向

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  硬件设备   │ ──> │  感知模块   │ ──> │  核心控制   │ ──> │  前端界面   │
└─────────────┘     └─────────────┘     └─────────────┘     └─────────────┘
       ^                 ^                 ^                 ^
       │                 │                 │                 │
       └─────────────────└─────────────────└─────────────────┘
```

### 3.2 核心交互流程

#### 3.2.1 系统启动流程
1. **硬件初始化**：平板、摄像头、投影仪启动
2. **系统初始化**：加载配置、初始化模块
3. **状态检测**：检测用户状态和环境信息
4. **界面加载**：显示主界面，准备交互

#### 3.2.2 认知训练流程
1. **游戏选择**：用户选择训练游戏
2. **游戏准备**：显示规则，校准脚部位置
3. **游戏进行**：
   - 系统投影游戏界面
   - 用户通过脚部交互
   - 系统实时检测和反馈
   - 动态调整难度
4. **游戏结束**：显示成绩，保存训练数据

#### 3.2.3 语音交互流程
1. **唤醒**：用户说出"阿康阿康"或点击语音按钮
2. **语音识别**：系统识别用户语音
3. **意图理解**：大语言模型分析用户意图
4. **执行响应**：
   - 回答用户问题
   - 执行请求操作
   - 跳转到相应页面
5. **语音反馈**：系统通过语音回复用户

#### 3.2.4 健康监测流程
1. **数据采集**：持续监测用户生理和行为状态
2. **数据处理**：分析和处理监测数据
3. **状态评估**：评估用户健康状态和趋势
4. **反馈建议**：生成健康建议和预警

### 3.3 实时通信

- **技术**：Socket.IO
- **通信内容**：
  - 系统状态更新
  - 游戏运行时数据
  - 感知数据更新
  - 语音状态更新
  - 页面导航指令
- **通信频率**：
  - 感知数据：500ms
  - 游戏运行时：100ms
  - 系统状态：1000ms

## 4. 技术实现细节

### 4.1 前端实现

#### 4.1.1 响应式设计
- **比例锁定**：使用CSS aspect-ratio: 16 / 10; 强制锁定平板显示比例
- **视口适配**：使用vh/vw单位和Flex布局，确保在不同设备上的显示效果
- **视频填充**：使用object-fit: cover; 确保视频充满容器

#### 4.1.2 游戏渲染
- **Canvas绘图**：使用HTML5 Canvas 2D API绘制游戏界面
- **动画循环**：requestAnimationFrame实现流畅动画
- **碰撞检测**：数学计算实现脚部位置与游戏元素的碰撞检测

#### 4.1.3 状态管理
- **全局状态**：使用Pinia管理全局状态
- **本地存储**：使用localStorage存储用户偏好
- **Socket通信**：实时接收后端状态更新

### 4.2 后端实现

#### 4.2.1 多线程处理
- **线程安全**：使用线程锁确保状态更新的线程安全
- **异步处理**：使用Flask-SocketIO的异步特性处理并发请求
- **并行计算**：视频处理和状态分析的并行计算

#### 4.2.2 视频处理
- **视频捕获**：使用OpenCV捕获视频流
- **图像处理**：使用OpenCV进行图像处理和分析
- **面部识别**：使用DeepFace进行面部表情分析
- **心率估计**：使用rPPG技术估计心率

#### 4.2.3 游戏逻辑
- **状态机**：使用状态机管理游戏状态
- **难度调整**：基于用户表现动态调整游戏难度
- **数据记录**：详细记录游戏过程和结果数据

#### 4.2.4 语音处理
- **语音唤醒**：使用关键词检测实现语音唤醒
- **语音识别**：使用Sherpa-ai进行语音识别
- **语音合成**：支持VITS和pytts两种TTS引擎

### 4.3 硬件集成

#### 4.3.1 摄像头集成
- **平板摄像头**：通过IP Webcam应用将视频流传输到电脑
- **外置摄像头**：通过USB连接直接捕获视频
- **视频同步**：多摄像头视频流的同步处理

#### 4.3.2 投影仪集成
- **投影校准**：通过摄像头校准投影区域
- **交互区域**：定义适合脚部操作的交互区域
- **视觉反馈**：通过颜色和动画提供视觉反馈
