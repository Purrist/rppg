### 1.2 核心原则

| 层级 | 职责 | 原则 |
|-----|------|------|
| **后端** | 唯一数据中心 | 所有状态都在这里管理，保存到JSON文件 |
| **前端** | 订阅和显示 | 不存储状态！只订阅后端更新，发送操作到后端 |
| **通信** | Socket.IO | 实时双向通信，后端广播状态变更 |

### 1.3 技术栈

| 层级 | 技术 | 用途 |
|-----|------|------|
| 前端 | Nuxt.js 4.2.2 + Vue 3.5 | UI框架 |
| 前端 | Socket.IO Client 4.8.3 | 实时通信（只订阅） |
| 后端 | Flask 3.1.2 | Web服务器 |
| 后端 | Flask-SocketIO 5.6.0 | WebSocket服务（广播状态） |
| 后端 | OpenCV 4.8.0 + MediaPipe 0.10.9 | 计算机视觉 |
| AI | Ollama + qwen2.5:3b | 本地大语言模型 |

```python
    def _load_config(self):
        """从文件加载配置"""
        try:
            if os.path.exists(self.config_file):
                with open(self.config_file, 'r', encoding='utf-8') as f:
                    saved = json.load(f)
                    self._state['aiMode'] = saved.get('aiMode', 'basic')
                    self._state['settings'] = {**self._state['settings'], **saved.get('settings', {})}
                    print(f'[SystemCenter] 配置已加载')
        except Exception as e:
            print(f'[SystemCenter] 加载配置失败: {e}')
    
    def _save_config(self):
        """保存配置到文件"""
        try:
            config = {
                'aiMode': self._state['aiMode'],
                'settings': self._state['settings']
            }
            with open(self.config_file, 'w', encoding='utf-8') as f:
                json.dump(config, f, ensure_ascii=False, indent=2)
        except Exception as e:
            print(f'[SystemCenter] 保存配置失败: {e}')
```

```javascript
export function initStore(socketInstance) {
  socket = socketInstance
  
  // ⭐ 监听后端状态更新
  socket.on('system_state', (data) => {
    updateState(data)  // 更新本地缓存
  })

  socket.on('game_update', (data) => {

      notifyListeners('game', data)
    }
  })
  
  // 监听感知数据
  socket.on('perception_update', (data) => {
    if (data) {
      state.value.perception = { ...state.value.perception, ...data }
```

```javascript
<script setup>
import { ref, onMounted, onUnmounted } from 'vue'
import { io } from 'socket.io-client'
import { initStore, subscribe, setAIMode, setDwellTime } from '../core/systemStore.js'

// Socket.IO 连接
let socket = null
let unsubscribe = null

// 确认时间（从后端同步）
const dwellTime = ref(2000)

// AI模式开关（从后端同步）
const aiCompanionEnabled = ref(false)

onMounted(() => {
  // 建立 Socket.IO 连接
  socket = io(backendUrl, { transports: ['polling', 'websocket'] })
  
  // ⭐ 初始化系统Store
  initStore(socket)
  
  socket.on('connect', () => {
    console.log('[settings] 后端已连接')
    // 请求当前状态
    socket.emit('get_system_state')
  })
  
  // ⭐ 订阅后端状态更新
  unsubscribe = subscribe((key, value, state) => {
    if (key === 'init' || key === 'state') {
      // 初始化或完整状态更新
      dwellTime.value = state.settings?.dwellTime || 2000
      aiCompanionEnabled.value = state.aiMode === 'companion'
    }
    if (key === 'dwellTime' || key === 'settings') {
      dwellTime.value = state.settings?.dwellTime || 2000
    }
    if (key === 'aiMode') {
      aiCompanionEnabled.value = value === 'companion'
    }
  })
})

onUnmounted(() => {
  if (unsubscribe) unsubscribe()
  if (socket) socket.disconnect()
})
```

## 四、数据流动分析

### 4.1 状态更新流程（后端→前端）

```
后端 SystemCenter 状态变更
    │
    ▼
_system_center.set_ai_mode('companion')
    │
    ├── 更新 _state['aiMode']
    ├── _save_config() 保存到JSON文件
    └── _broadcast() 广播到所有前端
            │
            ▼ Socket.IO emit('system_state', state)
            │
    ┌───────┴───────┐
    │               │
settings.vue   developer.vue
    │               │
subscribe()    subscribe()
回调收到更新    回调收到更新
    │               │
更新UI显示      更新UI显示
```


## 七、总结

### 7.1 核心设计原则

1. **后端是唯一的真相来源**
   - 所有状态都在 `SystemCenter` 管理
   - 保存到 `system_config.json` 持久化
   - 通过 Socket.IO 广播到所有前端

2. **前端只订阅和显示**
   - `systemStore.js` 只缓存后端状态
   - 所有操作发送到后端处理
   - 等待后端广播更新

3. **实时同步**
   - Socket.IO 实现双向实时通信
   - 状态变更立即广播到所有页面
   - 确保所有页面数据一致

### 7.2 添加新功能的步骤

1. **后端**:
   - 在 `SystemCenter` 添加新的状态字段
   - 添加 getter/setter 方法
   - 在 `app.py` 添加 Socket 事件处理

2. **前端**:
   - 在 `systemStore.js` 添加对应字段
   - 添加发送操作到后端的函数
   - 在页面中订阅并显示
