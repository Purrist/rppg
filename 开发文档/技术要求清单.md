## 一、硬件设备要求

### 1.1 计算设备

- **PC 或服务器**：运行后端系统，要求 CPU 4核以上、内存 8GB 以上，支持实时信号处理。
- **Android 平板**：作为系统前端界面，要求 2560×1600 分辨率、前置摄像头、麦克风、扬声器。

### 1.2 感知设备

- **毫米波雷达**：非接触式生理监测，支持心率、呼吸率检测，建议型号为 TI AWR1843 或类似。
- **平板前置摄像头**：用户面部表情、姿态、参与度采集，要求 720p/30fps。
- **地面交互摄像头**：投影交互脚部检测，要求 1080p/30fps、支持畸变校正。
- **投影仪**：地面投影，要求 1920×1080、3000流明、支持梯形校正。
- **麦克风阵列**：语音唤醒与识别，支持远距离拾音和降噪。

### 1.3 网络设备

- **无线局域网**：平板与 PC 通信，要求 Wi-Fi 5 或以上，无明显延迟。
- **USB 集线器**：连接多种外设，要求 USB 3.0 或以上。

## 二、后端技术栈

### 2.1 编程语言与框架

- **Python 3.10+**：后端主要开发语言，要求虚拟环境管理，确保依赖隔离。
- **FastAPI**：后端 Web 框架，用于构建 RESTful API，支持异步处理。
- **Uvicorn**：ASGI 服务器，用于部署 FastAPI 应用，支持多进程。

### 2.2 科学计算与信号处理

- **NumPy**：数值计算库，用于生理信号处理，要求掌握向量化运算。
- **Pandas**：数据分析库，用于数据统计、趋势分析，要求掌握数据清洗。
- **SciPy**：科学计算库，用于信号滤波、傅里叶变换等。
- **Matplotlib**：数据可视化库，用于生成健康报告图表。

### 2.3 多线程与异步编程

- **threading**：Python 标准库，用于语音识别、感知处理等后台任务。
- **asyncio**：异步 I/O 库，用于 WebSocket 通信、API 并发处理。

## 三、前端技术栈

### 3.1 基础框架

- **Vue 3**：前端主要框架，采用 Composition API，要求响应式数据管理。
- **Vite**：构建工具，用于开发环境热更新、生产环境打包，要求配置 16:10 比例适配。
- **TypeScript**：类型安全脚本语言，可选但推荐，提升代码质量。

### 3.2 UI 组件库

- **Element Plus**：Vue 3 组件库，用于按钮、表格、图表等，要求大字体、高对比度的适老化调整。
- **ECharts**：数据可视化库，用于健康数据图表、训练结果展示。
- **Tailwind CSS**（可选）：原子化 CSS 框架，用于快速构建响应式界面。

### 3.3 实时通信

- **Socket.IO**：WebSocket 客户端，用于与后端实时通信，支持自动重连。
- **Axios**：HTTP 请求库，用于 API 调用，要求错误处理和超时设置。

### 3.4 移动端适配

- **CSS aspect-ratio**：强制平板界面 16:10 比例，确保在电脑端缩放不变形。
- **Flex 布局**：自适应布局，处理不同设备分辨率。
- **大字体/大按钮**：最小字体 24px，最小按钮 48px×48px，符合适老化规范。

## 四、感知与监测技术

### 4.1 毫米波雷达生理监测

- **雷达数据解析**：解析毫米波雷达原始数据（如 TI AWR1843 输出的相位、幅度等）。
- **相位解卷绕**：处理相位跳变问题，确保相位数据连续性。
- **带通滤波**：去除心率、呼吸信号中的噪声，心率 0.5-3Hz，呼吸 0.1-0.5Hz。
- **峰值检测**：提取心跳、呼吸的峰值，用于频率估算。
- **PLV 计算**：心肺相位同步程度分析，值范围 0-1。
- **HRV 分析**：计算 RMSSD、SDNN、LF/HF 等时域频域指标。
- **动态阈值**：适应不同用户的基线水平。

### 4.2 计算机视觉

- **OpenCV**：计算机视觉基础库，用于图像处理、人脸检测、视频流捕获。
- **DeepFace**：面部表情识别库，支持 7 种基本情绪分类。
- **MediaPipe**（可选）：面部关键点、手部关键点、姿态估计。
- **MTCNN**：高精度人脸检测，支持侧脸、遮挡场景。

### 4.3 投影交互

- **投影校准**：四点标定法，将投影坐标与摄像头坐标映射。
- **透视变换**：处理投影畸变，确保交互区域准确。
- **脚部检测**：识别用户脚部位置，跟踪移动。
- **绿点追踪**：实时显示用户脚部位置，提供视觉反馈。
- **区域判定**：判断用户是否在正确交互区域内，支持矩形、圆形、三角形。

## 五、人工智能技术

### 5.1 语音处理

- **Sherpa-Onnx**：离线语音识别库，要求中文普通话识别。
- **VITS / Edge-TTS**：语音合成库，要求自然流畅的中文 TTS。
- **关键词唤醒**："阿康阿康"唤醒词检测，要求误唤醒率低。
- **静音检测**：判断用户是否在说话，避免无效处理。
- **语音降噪**：去除环境噪声，提高识别准确率。

### 5.2 大语言模型

- **通义千问 / Qwen**：本地运行的大语言模型，要求 7B 或以上参数。
- **提示词工程**：设计系统提示词，使模型理解角色定位、功能范围。
- **记忆机制**：短期记忆（对话历史）、长期记忆（用户偏好）。
- **工具调用**：模型调用系统功能，如页面跳转、游戏启动、紧急呼救。

### 5.3 认知训练

- **Go/No-Go 范式**：选择性注意训练，要求清晰的刺激呈现和反馈。
- **动态难度调整（DDA）**：基于用户表现调整游戏难度，维持心流状态。
- **心流状态估算**：结合 PLV、HRV、用户表现，判断是否进入最佳状态。
- **训练数据分析**：计算准确率、反应时、趋势，生成训练报告。

## 六、通信与数据传输技术

### 6.1 WebSocket 通信

- **双向实时通信**：后端向前端推送状态更新，前端向后端发送操作指令。
- **消息队列**：异步处理消息，避免阻塞，支持消息重发。
- **连接心跳**：定期发送心跳包，检测连接状态，自动重连。
- **序列化格式**：使用 JSON 传输数据，统一消息结构。

### 6.2 局域网通信

- **IP 配置**：PC 与平板在同一局域网内，设置固定 IP 或使用 mDNS。
- **HTTP/HTTPS**：前端访问后端 API，要求本地部署不暴露外网。
- **CORS 配置**：允许跨域请求，方便开发调试。

## 七、存储技术

### 7.1 本地数据库

- **SQLite**：轻量级关系型数据库，存储用户信息、健康数据、训练记录。
- **数据库设计**：用户表、健康数据表、训练记录表、记忆表等。

### 7.2 文件存储

- **JSON 文件**：配置文件、状态缓存、历史数据归档。
- **图像文件**：用户头像、训练截图、相册（可选）。

### 7.3 内存缓存

- **Python 字典/列表**：短期数据缓存，如当前对话、临时状态。
- **Redis**（可选）：高性能缓存，适合多用户场景。

---

## 八、部署与运维技术

### 8.1 环境管理

- **Conda 虚拟环境**：隔离 Python 依赖，避免版本冲突。
- **requirements.txt**：记录所有依赖包，便于快速部署。
- **环境变量**：敏感信息（如 API 密钥）通过环境变量管理。

### 8.2 日志与监控

- **Python logging**：系统日志记录，分级别（INFO、WARNING、ERROR）。
- **实时监控**：系统状态监控，如雷达连接、摄像头状态、内存使用。
- **错误恢复**：关键组件崩溃自动重启，保证系统可用性。

### 8.3 适老化设计实现

- **大字体高对比度**：符合老年用户视觉需求，对比度 ≥4.5:1。
- **清晰的视觉反馈**：按钮按下、操作成功等都有明确反馈。
- **语音引导**：操作步骤有语音提示，降低学习难度。
- **容错设计**：误操作有确认机制，重要操作防误触。

### 8.4 安全性与隐私

- **数据本地化**：所有数据存储在本地，不传输到云端。
- **无个人身份信息**：不收集身份证、手机号等敏感信息。
- **系统权限最小化**：只申请必要的系统权限。
