# RPPG 项目现状分析

## 1. 项目概述

RPPG（Real-time Physiological and Psychological Monitoring）是一个实时生理和心理监测系统，主要功能包括情绪识别、姿态估计、认知训练游戏等。项目采用前后端分离架构，集成了多种AI模型和实时视频处理技术。

## 2. 目录结构

```
rppg/
├── au/              # 情绪识别模块（OpenGraphAU）
├── backend/         # 后端服务（Flask + SocketIO）
├── demo/            # 测试脚本和示例
├── emo/             # 情绪识别相关实现（包含OpenFace-3.0）
├── frontend/        # 前端界面（Nuxt 4 + Vue 3）
├── 开发文档/        # 项目文档
├── .gitignore       # Git忽略文件配置
├── env_requirements.txt  # 环境依赖
└── 系统架构.md      # 系统架构文档
```

## 3. 核心模块

### 3.1 情绪识别模块（au/）

**功能**：基于面部动作单元（AU）的实时情绪检测

**主要文件**：
- `app.py`：情绪识别的主应用，包含完整的推理流程
- `au_net/`：OpenGraphAU模型实现
  - `MEFL.py`：主要模型文件
  - `resnet.py`：ResNet骨干网络
  - `swin_transformer.py`：Swin Transformer骨干网络
- `data/baselines.json`：情绪校准基线数据
- `templates/index.html`：情绪识别Web界面

**技术特点**：
- 使用MediaPipe进行面部关键点检测
- 使用SixDRepNet进行头部姿态估计
- 支持4种姿态（front/up/down/side）和3种情绪（calm/happy/sad）
- 实现了分姿态分情绪的校准系统
- 采用z-score归一化和状态机进行情绪判断

### 3.2 后端服务（backend/）

**功能**：系统核心服务，管理游戏、感知和语音模块

**主要文件**：
- `app.py`：后端主服务，基于Flask和SocketIO
- `core/`：系统核心模块
  - `system_core.py`：系统核心类
  - `core_agent.py`：AI代理
  - `training_analytics.py`：训练分析
- `games/`：游戏模块
  - `game_whack_a_mole.py`：打地鼠游戏
  - `processing_speed_game.py`：处理速度游戏
- `perception/`：感知模块
  - `perception_manager.py`：感知管理器

**技术特点**：
- 实时视频流处理
- WebSocket通信
- 多线程并发处理
- 游戏难度自动调整

### 3.3 前端界面（frontend/）

**功能**：用户交互界面，显示情绪状态和游戏界面

**主要文件**：
- `nuxt.config.ts`：Nuxt配置
- `pages/`：页面组件
  - `index.vue`：首页
  - `training.vue`：训练页面
  - `processing-speed.vue`：处理速度游戏页面
  - `whack-a-mole.vue`：打地鼠游戏页面
- `components/`：通用组件

**技术特点**：
- 基于Nuxt 4和Vue 3
- 响应式设计
- Socket.io实时通信
- 现代化UI界面

### 3.4 情绪识别辅助模块（emo/）

**功能**：情绪识别的替代实现和参考

**主要文件**：
- `OpenFace-3.0/`：OpenFace实现
  - `model/AU_model.py`：AU检测模型
  - `weights/`：模型权重
- `src/`：自定义情绪处理模块
  - `emotion_mapper.py`：情绪映射器
  - `feature_extractor.py`：特征提取器

## 5. 系统流程

1. **初始化**：启动后端服务和前端界面
2. **视频采集**：通过摄像头获取实时视频流
3. **预处理**：面部检测和关键点提取
4. **姿态估计**：使用SixDRepNet估计头部姿态
5. **AU检测**：使用OpenGraphAU检测面部动作单元
6. **情绪识别**：基于AU和姿态进行情绪判断
7. **游戏交互**：根据情绪状态调整游戏难度
8. **数据存储**：保存训练数据和情绪记录

## 9. 关键API

### 9.1 情绪识别API

- `GET /api/status`：获取当前情绪状态
- `POST /api/calibrate/start`：开始校准
- `POST /api/calibrate/stop`：停止校准
- `POST /api/calibrate/delete`：删除校准数据
- `POST /api/calibrate/delete_all`：删除所有校准数据
- `GET /api/config`：获取配置
- `POST /api/config`：更新配置

### 9.2 后端服务API

- `GET /`：前端界面
- `GET /api/system/status`：系统状态
- `GET /api/games/list`：游戏列表
- `POST /api/games/start`：开始游戏
- `POST /api/games/stop`：停止游戏

## 10. 项目价值

1. **心理监测**：实时监测用户情绪状态
2. **认知训练**：通过游戏提升认知能力
3. **个性化体验**：根据情绪状态调整游戏难度
4. **数据收集**：积累情绪和认知训练数据
5. **技术整合**：集成多种AI模型和实时处理技术

## 11. 总结

RPPG项目是一个集成了情绪识别、姿态估计和认知训练的综合系统，采用现代化的技术栈和架构设计。项目已实现核心功能，具有良好的扩展性和应用前景。当前需要解决的主要问题是Git大文件推送限制和系统稳定性优化。

未来可考虑扩展更多情绪类型、增强多用户支持、优化模型性能，以及开发更多类型的认知训练游戏，进一步提升系统的实用性和用户体验。
