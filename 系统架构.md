## 2. 数据流分析

### 2.1 游戏启动流程

```
用户点击游戏卡片
    │
    ▼
learning.vue: showGameInfo(gameName)
    │
    ▼
用户点击"开始游戏"
    │
    ▼
learning.vue: confirmStartGame()
    │
    ├── 从 localStorage 读取 dwellTime
    │
    ▼
socket.emit('game_control', { action: 'ready', game: 'processing_speed', dwell_time: 2000 })
    │
    ▼
后端: app.py handle_game_control()
    │
    ├── 更新 PROCESSING_SPEED_CONFIG.dwell_time
    │
    ▼
game_manager.set_ready('processing_speed')
    │
    ├── 如果 game_id != current_game_id: create_game(game_id)
    │   │
    │   └── 创建新的 ProcessingSpeedGame 实例
    │       └── 传入 PROCESSING_SPEED_CONFIG
    │
    ▼
ProcessingSpeedGame.set_ready()
    │
    ├── state.status = 'READY'
    │
    ▼
socket.emit('game_update', { game_id: 'processing_speed', status: 'READY', ... })
    │
    ▼
前端: projection.vue 收到 game_update
    │
    ├── 更新 game.value.status = 'READY'
    │
    ▼
watch: game.value.status 变化
    │
    ▼
gameState.value = 'READY'
    │
    ▼
显示准备圆圈，等待用户进入
```

### 2.2 游戏进行流程

```
用户进入准备圆圈3秒
    │
    ▼

    │
    ▼
socket.emit('game_control', { action: 'start' })
    │
    ▼
后端: game_manager.start_game()
    │
    ▼
ProcessingSpeedGame.start_game()
    │
    ├── state.status = 'PLAYING'
    │
    ▼
ProcessingSpeedGame._on_start()
    │
    ▼
_start_interval()  // 开始作答间隔
    │
    ▼
游戏主循环 (main_worker 每33ms调用一次)
    │
    ▼
ProcessingSpeedGame.update()
    │
    ├── 检查间隔是否结束
    │   │
    │   └── 结束: _start_question()  // 开始新题目
    │       │
    │       ├── 生成题目 (Go/No-Go 或 选择反应)
    │       │
    │       └── _emit_question()  // 发送题目到前端
    │
    ▼
前端: ProcessingSpeedGame.vue 收到题目
    │
    ├── 显示8个区域 (根据 zones 状态)
    │
    ├── 显示指令 (如"踩绿色！")
    │
    └── 显示倒计时进度条
    │
    ▼
用户站在某个区域
    │
    ▼
checkDwell() 每50ms检测
    │
    ├── 计算停留时间
    │
    ├── 更新 zoneProgress[i] (0-100%)
    │
    └── 绘制进度圈
    │
    ▼

    │
    ▼

    │
    ▼

    │
    ▼

    │
    ▼
后端: ProcessingSpeedGame._handle_answer(zone_id)
    │
    ├── 判断对错
    │
    ├── 更新分数
    │
    └── _emit_feedback()
    │
    ▼
_start_interval()  // 进入间隔期
```

## 3. 发现的问题

### 问题1: 游戏类型切换问题

**现象**: 点击处理速度训练，但开始的是打地鼠游戏

**原因分析**:
1. 在 `projection.vue` 的 `game_update` 处理中，判断游戏类型的逻辑有问题：
```javascript
const gameId = data.game_id || ''
if (gameId === 'processing_speed' || data.module) {
  currentGame.value = 'processing_speed'
}
```
问题：`data.module` 在打地鼠游戏中也可能存在，导致误判。

2. 更根本的问题：`currentGame` 初始值是 `'whack_a_mole'`，如果在收到第一个 `game_update` 之前就开始渲染，会显示错误的游戏。

### 问题2: 确认时间设置不生效

**现象**: 在 setting.vue 调整确认时间，游戏没有使用这个变量

**原因分析**:
1. `setting.vue` 将 `dwellTime` 保存到 `localStorage`
2. `learning.vue` 在发送 `ready` 时从 `localStorage` 读取并传递给后端
3. 后端 `app.py` 更新 `PROCESSING_SPEED_CONFIG.dwell_time`
4. **问题**: `create_game()` 在 `set_ready()` 中被调用时，配置对象已经被传入，但 `ProcessingSpeedGame` 在 `__init__` 中读取 `self.config.dwell_time`，此时配置还没有被更新

**数据流问题**:
```
learning.vue 发送 dwell_time ──► app.py 更新 CONFIG
                                       │
                                       ▼
                              set_ready() 被调用
                                       │
                                       ▼
                              create_game() 创建实例
                                       │
                                       ▼
                              __init__() 读取 config.dwell_time
                              (此时读取的是旧值！)
```

### 问题3: 进度圈和进度条不显示

**现象**: projection 上没有"踩绿色"这样的题目，也没有倒计时进度条，站在八个区域里也不出现0-100的进度圈

**原因分析**:
1. **题目不显示**: `ProcessingSpeedGame.vue` 组件只在 `gameState === 'PLAYING' && currentGame === 'processing_speed'` 时显示，但 `currentGame` 可能没有被正确设置

2. **进度圈不显示**: `checkDwell()` 函数检测停留，但可能存在以下问题：
   - `props.footPosition` 坐标系不匹配
   - `zoneProgress` 是响应式对象，但使用方式可能有问题
   - 绘制函数中 `zoneProgress.value[i]` 可能为 undefined

3. **进度条不显示**: `remainingTime` 和 `answerTime` 可能没有被正确更新

### 问题4: 一直在弹错误和正确

**现象**: 一直在弹错误/正确，没有间隔期

**原因分析**:
1. 可能是 `_start_interval()` 和 `_start_question()` 之间的逻辑有问题
2. 间隔时间太短（1.5秒），用户感知不到
3. 可能是前端反馈显示逻辑有问题，`feedback` 没有被正确清除

### 问题5: 难度调整写死在游戏py里

**现象**: 游戏难度的调整是由游戏自己决定的，不是由 core 统一管理

**原因分析**:
当前 `ProcessingSpeedGame._adjust_difficulty()` 直接在游戏类中实现，根据 `self.recent_trials` 计算准确率并调整难度。

应该改为：
1. 游戏只上报表现数据（正确/错误、反应时间）
2. core 的 `SystemStateManager` 或专门的难度管理器根据用户整体状态决定难度
3. 游戏接收难度调整指令

## 4. 解决方案

### 解决方案1: 修复游戏类型切换

在 `projection.vue` 中，严格根据 `game_id` 判断游戏类型：
```javascript
// 只使用 game_id 判断，不使用 module
if (gameId === 'processing_speed') {
  currentGame.value = 'processing_speed'
} else if (gameId === 'whack_a_mole') {
  currentGame.value = 'whack_a_mole'
}
```

### 解决方案2: 修复确认时间设置

方案A: 延迟创建游戏实例，等配置更新后再创建
方案B: 在 `_on_ready()` 中重新读取配置
方案C: 将 dwell_time 作为参数传递给游戏，而不是通过配置

推荐方案C，最简单直接：
```python
# app.py
game_manager.set_ready(game_id, dwell_time=dwell_time)

# games_manager.py
def set_ready(self, game_id: str = None, dwell_time: int = None):
    if game_id:
        if game_id != self._current_game_id:
            self.create_game(game_id, dwell_time=dwell_time)
    # ...
```

### 解决方案3: 修复进度圈和进度条

1. 确保 `ProcessingSpeedGame.vue` 正确接收和解析 `gameState`
2. 修复 `zoneProgress` 的响应式问题
3. 添加调试日志，追踪数据流

### 解决方案4: 修复难度调整

创建难度管理器，由 core 统一管理：
```python
# core/difficulty_manager.py
class DifficultyManager:
    def __init__(self):
        self.user_performance = {}
    
    def record_trial(self, game_id: str, correct: bool, rt: float):
        # 记录表现
        pass
    
    def get_difficulty_level(self, game_id: str) -> int:
        # 根据用户整体表现决定难度
        pass
```

## 5. 关键文件关系

```
frontend/pages/learning.vue
    ├── 调用 socket.emit('game_control', { action: 'ready', game, dwell_time })
    └── 跳转到 /training

frontend/pages/training.vue
    ├── 显示游戏控制界面
    └── 调用 socket.emit('game_control', { action: 'start' })

frontend/pages/projection.vue
    ├── 接收 socket.on('game_update', ...)
    ├── 根据 currentGame 显示不同组件
    │   └── ProcessingSpeedGame.vue (处理速度训练)
    └── 绘制绿点、处理用户位置

frontend/components/ProcessingSpeedGame.vue
    ├── 接收 gameState (包含 question, remaining_time 等)
    ├── 绘制8个区域
    ├── 检测用户停留 (checkDwell)
    ├── 绘制进度圈
    └── 发送 action 到父组件

backend/app.py
    ├── handle_game_control() 处理游戏控制
    ├── handle_game_action() 处理游戏动作
    └── 初始化 GameManager

backend/games/games_manager.py
    ├── register() 注册游戏
    ├── create_game() 创建游戏实例
    ├── set_ready() 准备游戏
    └── handle_action() 转发动作到游戏

backend/games/games_base.py
    └── GameBase 抽象基类
        ├── set_ready() -> _on_ready()
        ├── start_game() -> _on_start()
        ├── update() -> _on_update()
        └── handle_action() -> _on_action()

backend/games/processing_speed_game.py
    └── ProcessingSpeedGame(GameBase)
        ├── _on_ready() 初始化
        ├── _on_start() 开始游戏
        ├── _on_update() 更新循环
        ├── _on_action() 处理动作
        ├── _start_interval() 开始间隔
        ├── _start_question() 开始题目
        ├── _handle_answer() 处理答案
        └── _adjust_difficulty() 调整难度 (需要重构)
```
