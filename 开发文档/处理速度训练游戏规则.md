# 处理速度训练游戏规则

> 基于30+篇认知训练文献重新设计，适配地面投影交互

---

## 一、核心概念定义

### 1.1 确认时间（Dwell Time）

**定义**：用户在识别区域内停留以确认选择所需的时间。

**特性**：
- **固定性**：单次游戏过程中保持不变，不参与难度调整
- **可调范围**：1500ms - 4000ms（用户在设置页面调整）
- **默认值**：2000ms
- **设计原理**：
  - 防止误触，确保意图明确
  - 适应老年人反应和移动速度
  - 用户可根据自身情况调整

**与作答时间的关系**：
```
作答时间 ≥ 确认时间 + 1000ms
```

---

### 1.2 作答时间（Answer Time）

**定义**：每道题的答题时限，从题目呈现到必须完成确认的时间。

**特性**：
- **动态性**：根据难度等级动态调整
- **计算公式**：`作答时间 = 确认时间 + 难度时间增量`
- **难度时间增量范围**：1000ms - 5000ms
- **视觉表现**：倒计时进度条从绿色渐变为红色

**超时处理**：
- 视为答题失败
- 扣除5分
- 进入下一题

---

### 1.3 识别区域（Zones）

**定义**：地面投影上的圆形交互区域，用户通过站在区域内完成答题。

**布局**（640×360设计坐标系）：
```
    [1]      [2]      [3]      [4]
   (80,120) (226,120) (372,120) (518,120)
    
    [5]      [6]      [7]      [8]
   (80,270) (226,270) (372,270) (518,270)
```

**区域参数**：
- 半径：70px
- 间距：水平约146px，垂直约150px

---

### 1.4 干扰项（Distractors）

**定义**：增加任务难度的非目标元素。

**类型**：
1. **No-Go刺激**：红色区域，需要抑制响应
2. **相似颜色**：与目标颜色相似的其他颜色
3. **多激活区域**：同时激活多个区域增加选择难度

## 二、游戏模块

### 2.1 模块一：Go/No-Go 反应控制训练

**训练目标**：反应抑制能力

**游戏规则**：

```
Go条件（需要响应）：
1. 目标区域显示绿色
2. 屏幕显示指令："踩绿色！"
3. 用户需在作答时间内踩踏绿色区域
4. 停留满确认时间后判定为正确
5. 得分：+10分 + 反应时间奖励

No-Go条件（需要抑制）：
1. 目标区域显示红色
2. 屏幕显示指令："别踩红色！"
3. 用户需要踩其他任意非红色区域
4. 停留满确认时间后判定为正确
5. 得分：+15分（抑制成功的额外奖励）

错误情况：
1. Go条件未响应：超时错误，-5分
2. No-Go条件踩踏红色：抑制失败，-10分
3. 踩踏错误区域：选择错误，-5分
```

**Go/No-Go比例与难度**：

| 难度等级 | Go比例 | No-Go比例 | 说明 |
|----------|--------|-----------|------|
| 1-2级 | 80-90% | 10-20% | Go占多数，抑制需求低 |
| 3-5级 | 60-70% | 30-40% | 平衡的Go/No-Go比例 |
| 6-8级 | 40-50% | 50-60% | 高抑制需求 |

---

### 2.2 模块二：选择反应训练

**训练目标**：决策速度和视觉搜索能力

**游戏规则**：

```
基本流程：
1. 屏幕显示目标颜色指令："踩蓝色！"
2. 2-8个区域被激活，显示不同颜色
3. 用户需要找到并踩踏目标颜色区域
4. 停留确认时间后判定
5. 正确：+10分 + 反应时间奖励
6. 错误：-5分

颜色配置：
- 低难度：2-3种颜色，差异明显（红、蓝、绿）
- 中难度：4-5种颜色，部分相似
- 高难度：6-8种颜色，多组相似色

相似颜色组：
- 红色系：红色、橙色、粉色
- 蓝色系：蓝色、青色、紫色
- 绿色系：绿色、黄绿色
- 黄色系：黄色、金色
```

**选择数量与难度**：

| 难度等级 | 激活区域数 | 颜色种类 | 干扰项 |
|----------|------------|----------|--------|
| 1-2级 | 2-3个 | 2-3种 | 无 |
| 3-4级 | 4-5个 | 4-5种 | 1-2个相似色 |
| 5-6级 | 6-7个 | 5-6种 | 2-3个相似色 |
| 7-8级 | 8个 | 6-8种 | 3-4个相似色 |

---

### 2.3 模块三：序列反应训练（SRTT）

**训练目标**：序列学习和程序记忆

**游戏规则**：

```
基本流程：
1. 屏幕显示序列提示："2 → 5 → 3"
2. 用户按顺序踩踏区域2、区域5、区域3
3. 每步需要停留确认
4. 显示序列进度："1/3"、"2/3"、"3/3"
5. 完成整个序列后获得额外奖励

序列长度：
- 低难度：3步序列
- 中难度：4-5步序列
- 高难度：6-8步序列

序列类型：
1. 显式序列
   - 屏幕显示完整序列
   - 用户按显示顺序执行

2. 隐式序列
   - 仅显示当前目标区域
   - 用户需要记住序列模式
   - 同一序列重复5次

计分规则：
- 每步正确：+10分
- 完成序列奖励：+50分
- 步骤错误：序列重置，-5分
```

**内隐学习评估**：
```
同一序列重复5次，记录每次反应时间
反应时间逐渐缩短表明内隐学习发生

学习效果计算：
学习效果 = (第1次平均RT - 第5次平均RT) / 第1次平均RT × 100%
```

## 三、游戏流程

### 3.1 四状态流程

```
IDLE（待机）→ READY（预备）→ PLAYING（游戏中）→ SETTLING（结算）
                    ↑___________________________________|
```

**IDLE状态**：
- 显示粒子效果背景
- 等待游戏启动

**READY状态**：
- 显示准备圆圈
- 平板显示游戏规则
- 玩家站入圆圈3秒开始

**PLAYING状态**：
- 执行游戏模块
- 实时显示得分和倒计时
- 单题倒计时条递减

**SETTLING状态**：
- 显示最终得分
- 显示准确率
- 5秒后自动返回READY

---

### 3.2 单题流程

```
时间轴：
0ms        1000-2000ms      3000-8000ms        确认时间后      确认时间+1000ms
│              │                 │                 │                 │
│   间隔期     │    题目呈现     │    用户响应     │     反馈       │   下一题
│   (空白)     │   (刺激显示)    │   (移动+停留)   │   (对错显示)   │   (间隔)
│              │                 │                 │                 │
▼              ▼                 ▼                 ▼                 ▼

详细流程：

1. 间隔期（1000-2000ms）
   - 所有区域显示为灰色
   - 无进度环响应
   - 屏幕显示"准备..."

2. 题目呈现（作答时间开始）
   - 随机选择区域激活
   - 显示指令（如"踩绿色！"）
   - 作答倒计时条开始递减

3. 用户响应
   - 用户移动到目标区域
   - 进度环开始增长（0-100%）
   - 停留满确认时间后触发确认

4. 反馈显示（1000ms）
   - 正确：绿色✓ + "+10分"
   - 错误：红色✗ + "-5分"
   - 显示反应时间

5. 进入下一题间隔期
```

## 四、计分规则

### 4.1 基础计分

| 情况 | 得分 | 说明 |
|------|------|------|
| Go正确 | +10分 | 正确踩踏绿色区域 |
| No-Go正确 | +15分 | 正确踩非红色区域 |
| 选择反应正确 | +10分 | 正确选择目标颜色 |
| 序列反应正确（每步） | +10分 | 正确完成序列的一步 |
| 序列完成奖励 | +50分 | 正确完成整个序列 |
| Go错误 | -5分 | 未踩踏绿色区域 |
| No-Go错误 | -10分 | 错误踩踏红色区域 |
| 选择反应错误 | -5分 | 选择错误颜色 |
| 序列反应错误 | -5分 | 序列步骤错误 |
| 超时 | -5分 | 未在作答时间内完成 |

### 4.2 反应时间奖励

```
反应时间奖励计算：

基础分 × (1 + 时间奖励系数)

时间奖励系数 = (作答时间 - 实际反应时间) / 作答时间

示例：
- 作答时间：5000ms
- 实际反应时间：2000ms
- 时间奖励系数 = (5000 - 2000) / 5000 = 0.6
- 最终得分 = 10 × (1 + 0.6) = 16分

上限：基础分 × 2（最高翻倍）
```

### 4.3 连续正确奖励

| 连续正确次数 | 奖励倍数 |
|--------------|----------|
| 3次 | ×1.2 |
| 5次 | ×1.5 |
| 10次 | ×2.0 |
| 15次以上 | ×2.5 |

## 六、难度调整策略

### 6.1 调整原则

**最佳训练区间理论**：正确率 60-90% 是最佳训练区间

| 正确率区间 | 处理策略 |
|------------|----------|
| < 60% | 任务过难，应降低难度 |
| 60-90% | 最佳区间，维持当前难度 |
| > 90% | 任务过易，应提高难度 |

---

### 6.2 调整规则

**升级条件**（需同时满足）：
1. 连续3个窗口正确率 > 85%，或连续5个窗口正确率 > 80%
2. 平均反应时间 < 基准RT × 0.85
3. 距离上次升级已过10个试次

**降级条件**（满足任一）：
1. 连续2个窗口正确率 < 50%（立即降级）
2. 连续3个窗口正确率 < 60%
3. 平均反应时间 > 作答时间 × 0.8 且正确率 < 70%
4. 距离上次降级已过5个试次

**滑动窗口机制**：
- 窗口大小：5个试次
- 评估频率：每完成5个试次评估一次

---

### 6.3 各模块特定调整

**Go/No-Go模块**：
- 升级：Go比例降低5%，作答时间减少500ms，激活区域数+1
- 降级：Go比例提高5%，作答时间增加500ms，激活区域数-1

**选择反应模块**：
- 升级：激活区域数+1，颜色种类+1，作答时间减少500ms，增加相似色干扰
- 降级：激活区域数-1，颜色种类-1，作答时间增加500ms，减少相似色干扰

**序列反应模块**：
- 升级：序列长度+1，作答时间减少500ms，从显式切换到隐式序列
- 降级：序列长度-1，作答时间增加500ms，从隐式切换到显式序列

## 八、评估指标

### 8.1 即时效果指标

| 指标 | 测量方法 | 预期改善 |
|------|----------|----------|
| 处理速度 | 平均反应时间 | 减少10-20% |
| 反应抑制 | No-Go抑制率 | 提高10-15% |
| 选择准确率 | 正确选择比例 | 提高5-10% |
| 序列学习 | 序列RT变化 | 减少15-25% |

### 8.2 单次游戏评估

```
基础统计：
- 总得分
- 总正确率
- 平均反应时间
- 最高连续正确数

模块统计：
- Go/No-Go模块：正确率、Go-RT、No-Go抑制率
- 选择反应模块：正确率、平均RT、选择分布
- 序列反应模块：正确率、学习曲线

难度分析：
- 起始难度等级
- 最终难度等级
- 难度调整次数
- 各难度停留时间
```
