## 一、系统概述与设计理念

### 1.1 项目背景与目标

本系统为工业设计专业毕业设计项目，研究主题为"结合健康监测与认知训练的老年陪伴机器人交互体验设计"。系统旨在构建一个面向机构养老场景的多模态交互陪伴机器人原型，具备"投影式认知交互 + 非接触式健康监测 + 情感陪伴机制"的联动框架，产出基于老人状态感知的自适应交互系统。

### 1.2 核心设计理念

#### 1.2.1 "活系统"理念

区别于传统的写死接口、固定游戏、简单推荐、被动响应的系统，本系统追求构建一个会自己"生长、感知、思考、主动做事"的活系统。满足以下五条标准：

1. **能感知环境**：老人在看什么、听什么、情绪如何、累不累
2. **能自己理解内容**：看懂西游记，知道人物、剧情、关系
3. **能自己产生目标**：我现在应该给老人做什么训练
4. **能自己调用工具**：去搜图、搜音乐、截片段、生成游戏
5. **能持续进化**：越用越懂老人，越用越聪明

#### 1.2.2 三大能力联动

系统中的三项核心能力构成闭环，而非独立存在：

- **健康监测**：提供生理与行为状态线索
- **情感陪伴**：提供长期互动与情绪反馈机制
- **认知训练**：作为主要交互任务，并根据状态动态调整

#### 1.2.3 投影交互优先

系统的核心交互形式为投影交互：
- 使用投影仪将认知训练界面投影到地面
- 老人通过**脚步 / 站位 / 停留**与投影界面交互
- 同时，可以在平板上通过传统的点击交互

#### 1.2.4 无感监测

视频识别与显示必须高效、流畅、准确，特别要解决侧脸、低头等非正脸/低光照、多人等复杂环境变化场景下的识别失败问题，实现真正的"无感监测"。

### 2.2 架构设计原则

#### 2.2.1 单一真相源 (Single Source of Truth)

SystemCore作为整个系统的唯一状态中心，所有状态变更必须通过SystemCore，状态变更自动广播到所有前端。这一设计确保了：
- 状态一致性：所有组件看到的状态是同步的
- 可预测性：状态变化路径清晰可追溯
- 可调试性：便于排查状态相关问题

#### 2.2.2 事件驱动架构

系统采用事件驱动架构，通过Socket.IO实现前后端实时通信：
- 前端通过事件触发后端动作
- 后端通过事件通知前端状态变化
- 支持多客户端同时订阅状态

#### 2.2.3 模块化设计

各功能模块独立设计，通过明确的接口进行交互：
- 游戏系统：支持多种认知训练游戏，易于扩展
- 感知系统：支持多种感知数据源，可插拔
- AI代理：可替换不同大模型实现

### 3.3 空间交互投影技术

#### 3.3.1 投影校准

系统通过透视变换(Perspective Transform)实现投影区域与摄像头视野的对应：
- 定义投影区域的四个角点
- 通过OpenCV计算透视变换矩阵
- 将摄像头坐标系中的脚部位置映射到投影坐标系

#### 3.3.2 脚部检测与跟踪

- 使用MediaPipe Pose进行人体骨骼检测
- 提取左脚踝(27)和右脚踝(28)关键点
- 计算两脚中心点作为交互位置
- 使用平滑滤波器减少抖动

#### 3.3.3 确认机制

老人站在目标区域内停留一定时间(默认2秒)即视为"确认"：
- 进度圈从0%到100%表示确认进度
- 确认时间可在设置中调整(500ms-10000ms)
- 支持动态难度调整确认时间

#### 4.2.3 线程安全设计

SystemCore使用线程锁保护状态访问：
```python
self._lock = threading.RLock()  # 可重入锁
self._listeners_lock = threading.Lock()  # 监听器锁
```

所有状态访问都通过锁保护：
```python
def get_state(self) -> Dict[str, Any]:
    """获取完整状态（线程安全）"""
    with self._lock:
        return self._state.copy()
```

#### 4.2.4 状态广播机制

SystemCore支持状态订阅模式，状态变化自动广播：
```python
def _broadcast(self, force=False, important_fields=None):
    """广播状态到所有前端"""
    with self._lock:
        state_copy = self._state.copy()
    
    # 通知监听器
    for listener in listeners:
        listener(state_copy)
    
    # Socket广播
    if self.socketio:
        self.socketio.emit('system_state', state_copy)
```

#### 4.3.3 已实现的认知训练游戏

**1. 趣味打地鼠 (Whack-A-Mole)**
- **训练目标**：手眼协调与反应速度
- **交互方式**：地鼠随机从三个洞中出现，站在对应地鼠洞上停留确认时间即可击中
- **计分规则**：击中得10分，错过扣5分
- **难度调整**：根据地鼠停留时间和出现间隔

**2. 处理速度训练 (Processing Speed Training)**
基于ACTIVE研究的经典认知训练范式，包含三个模块：

- **Go/No-Go模块（反应控制）**：
  - 绿色区域出现时要快速踩踏
  - 红色区域出现时要忍住不踩
  - 训练反应控制和抑制能力

- **选择反应模块（决策速度）**：
  - 根据提示踩踏对应颜色的区域
  - 颜色选项2-5个，基于Hick定律设计
  - 训练决策速度

- **序列学习模块（序列学习）**：
  - 按顺序踩踏亮起的区域
  - 完成整个序列
  - 训练序列学习能力

**难度分级系统（8级）**：
| 等级 | 作答额外时间 | 激活区域数 | 颜色数 |
|------|-------------|-----------|--------|
| 1    | +5.0s       | 3         | 2      |
| 2    | +4.5s       | 4         | 2      |
| 3    | +4.0s       | 5         | 3      |
| 4    | +3.5s       | 6         | 3      |
| 5    | +3.0s       | 6         | 4      |
| 6    | +2.5s       | 7         | 4      |
| 7    | +2.0s       | 7         | 5      |
| 8    | +1.5s       | 8         | 5      |

#### 4.3.4 动态难度调整 (DDA)

系统基于用户表现实时调整任务难度：
- **评估周期**：每5个试次评估一次
- **升级条件**：正确率>90%且反应时间快
- **降级条件**：正确率<60%
- **目标区间**：维持60-90%正确率的最佳训练区间

```python
}
```

#### 4.4.3 三维状态评估体系

系统从三个维度评估用户状态：

**1. 身体负荷 (Physical Load)**：
- 心率因子（参考Karvonen公式）
- 运动强度
- 姿态稳定性
- 疲劳迹象

**2. 认知负荷 (Cognitive Load)**：
- 反应时间
- 错误率
- 犹豫次数
- 注意力稳定性

**3. 参与意愿 (Engagement)**：
- 正面情绪
- 主动性
- 坚持性

### 4.5 AI代理系统 (core_agent.py)

## 六、核心功能模块详解

### 6.1 健康监测模块

- **疲劳度评估**：综合心率、眨眼率、注意力等指标

#### 6.1.2 情绪状态监测

- **面部表情识别**：使用DeepFace分析情绪
- **情绪维度**：valence（正负）和arousal（激活度）
- **情绪趋势**：记录情绪变化趋势

#### 6.1.3 参与度评估

- **注意力评分**：基于眼部状态和头部姿态
- **互动频率**：记录用户与系统的交互次数
- **坚持性**：记录训练时长和完成度

### 6.2 认知训练模块

#### 6.2.1 训练游戏设计原则

基于ACTIVE研究等文献，遵循以下原则：
- **难度适配**：动态调整难度，维持最佳挑战区间
- **即时反馈**：立即给出正确/错误反馈
- **渐进式**：从简单到复杂，循序渐进
- **多维度**：覆盖反应速度、注意力、记忆、执行功能等

#### 6.2.2 训练效果评估

- **反应时间**：记录每次反应的时间
- **准确率**：统计正确率
- **进步曲线**：展示长期训练效果
- **认知域覆盖**：评估不同认知域的训练效果

### 6.3 情感陪伴模块

#### 6.3.1 阿康助手

## 七、系统创新点

### 7.1 具身智能交互设计

#### 7.1.1 投影交互创新

**创新点**：将认知训练界面投影到地面，老人通过脚步交互

**优势**：
- **低门槛**：无需学习复杂操作，自然站立移动即可
- **全身参与**：身体活动与认知训练结合
- **社交性**：多人可同时参与
- **沉浸感**：大尺寸投影提供沉浸体验

**技术实现**：
- 透视变换实现摄像头与投影坐标对应
- MediaPipe Pose实时骨骼检测
- 平滑滤波减少抖动
- 确认机制防止误触

#### 7.1.2 多模态感知融合

**创新点**：融合平板摄像头和外置摄像头数据，构建完整用户画像

**感知维度**：
- **生理层**：心率、HRV、疲劳度
- **情绪层**：面部表情、情绪趋势
- **行为层**：身体姿态、活动量、交互频率
- **认知层**：反应时间、准确率、注意力

**融合方法**：
- 多源数据时间对齐
- 加权融合算法
- 趋势分析而非单点判断

### 7.2 自适应认知训练系统

#### 7.2.1 动态难度调整 (DDA)

**创新点**：基于用户实时表现动态调整难度，维持最佳训练区间

**算法特点**：
- **多尺度评估**：单题、单次训练、短期趋势、长期趋势
- **智能阈值**：60-90%正确率为最佳区间
- **渐进调整**：每次最多调整1级，避免剧烈变化
- **最佳难度推荐**：基于历史数据推荐初始难度

**文献依据**：
- ACTIVE研究显示正确率60-90%为最佳训练区间
- 动态难度调整能显著提升训练持续性

#### 7.2.2 基于状态的训练调节

**创新点**：根据用户身心状态动态调整训练参数

**调节策略**：
| 状态 | 调节策略 |
|------|----------|
| 疲劳度高 | 降低难度、缩短时长、增加休息提示 |
| 注意力低 | 简化任务、增加引导、降低干扰 |
| 情绪低落 | 增加鼓励、降低难度、推荐喜欢的内容 |
| 状态良好 | 适当增加挑战、尝试新内容 |

### 7.3 情感陪伴机制

#### 7.3.1 本地化大模型

**创新点**：使用本地大模型（Ollama + qwen2.5:3b）实现AI陪伴

**优势**：
- **隐私保护**：数据本地处理，不上传云端
