### 2024-03-08 v17 - 真正的具身智能体：能行动，不只是回答

**用户需求原文**：
> 我要的不是一个死的工具，是一个真正懂我的具身智能体
> 比如我问：我想看电影
> 回答不应该只是文字，而是已经跳转到了娱乐页面的电影栏目，并自动更改这个系统UI呈现几部这个老人最可能感兴趣的电影

**核心改变**：

1. **从问答到行动**：
   - 用户说"我想看电影" → 系统跳转到娱乐页面 + 展示推荐电影
   - 不是"问一句答一句"，而是真正做事

2. **LLM决定动作**：
   - 不是规则匹配，而是让LLM理解意图后自主决定要做什么
   - 返回动作类型：`navigate_and_recommend`、`navigate`、`play`、`game`

3. **前端接收推荐**：
   - 娱乐页面接收`akon_recommend`事件
   - 展示推荐内容卡片

**修改文件**：
- `/download/backend/akon_agent.py` - 重写，LLM返回动作决策
- `/download/backend/app.py` - 处理新的动作类型
- `/download/frontend/pages/entertainment.vue` - 接收推荐并展示

**动作类型**：
```json
// 导航+推荐
{
  "type": "navigate_and_recommend",
  "page": "/entertainment",
  "content": {
    "type": "movie",

  }
}

// 播放
{
  "type": "play",
  "content": {
    "type": "music",
    "items": ["贵妃醉酒", "霸王别姬"]
  }
}
```

**用户画像**：
```python
USER_MODEL = {

        "favorite_movies": ["地道战", "地雷战", "英雄儿女"],
```

   - 添加更多导航关键词

**修改文件**：

- `/download/backend/rppg_processor.py` - 改进算法，添加稳定性检测

### 2024-03-08 v14 - 视频流优化 + rPPG改进 + 对话修复

**用户需求原文**：
> 1.平板视频流可以显示了，但是太卡了！！要像外接摄像头一样流畅不卡顿
> 2.另外现在对人脸的识别不准，心率更是不准，受光线和移动的影响太大了
> 3.现在根本回答不了除了预设的三个选项之外的问题了，我想要的是快速思考快速回答！

**产出内容**：

1. **视频流卡顿修复**：
   - 将rPPG处理放到独立线程`rppg_worker()`
   - 视频流只从共享帧读取，不做处理
   - 使用线程锁保护共享帧

2. **rPPG算法改进**：
   - 使用MediaPipe人脸检测（更准确）
   - 使用CHROM算法（更鲁棒）
   - 添加跳帧处理（每3帧处理一次）
   - 提高置信度阈值

3. **对话修复**：
   - 添加更多简单模式匹配（电影、看、视频等）
   - 修复LLM返回空的问题
   - 添加错误处理和默认回复

**修改文件**：
- `/download/backend/app.py` - 添加rPPG独立线程，优化视频流
- `/download/backend/rppg_processor.py` - 使用MediaPipe+CHROM算法
- `/download/backend/akon_agent.py` - 修复对话返回空问题

**架构改进**：
```
之前：视频流 → rPPG处理 → 返回（卡顿）
现在：rPPG线程独立处理 → 共享帧 ← 视频流读取（流畅）
```
